    Amazon AWS #

    Starburst Galaxy and Starburst Enterprise platform (SEP) can be used with many components of Amazon AWS. You can deploy there, access data sources in Amazon AWS, and use some of the other features of Amazon AWS.

    Find all the relevant information in the following sections and guides.

    Starburst Galaxy #

    You can use Starburst Galaxy to run clusters on AWS, and configure catalogs to query numerous data sources on AWS.

    Starburst Enterprise #

    You can install SEP on Amazon Elastic Kubernetes Service (EKS), Amazon Elastic Compute Cloud (EC2) or through the AWS Marketplace, once you have established your enterprise account.

    To get started Starburst offers a trial license and help with your proof of concept.

    Find more guidance in our Try Starburst Enterprise section.

    SEP on Amazon EKS #

    Amazon Elastic Kubernetes Service (EKS) is certified to work with SEP.

    More information is available in our dedicated EKS section.

    SEP on Amazon EC2 #

    As a user of Amazon EC2 instances you can manage your SEP cluster with the same AMI and CloudFormation Template as AWS Marketplace users.

    Alternatively, you can manage a RPM or tarball-based installation with Starburst Admin.

    SEP on AWS Marketplace #

    SEP is available on AWS marketplace.

    Documentation #

    You are currently viewing the AWS section of our user guides. In the left-hand navigation are guides for AWS-specific topics. We also have user guides tailored toward the different types of Starburst users: data consumers, data engineers and platform administrators.

    Our comprehensive reference documentation is available for all supported releases.

    Support for SEP #

    Starburst customers using EKS or EC2 enjoy:

    • Dedicated resources for your organization
    • 24/7 expert support
    • Rapid-response SLAs
    • Access to knowledge base articles and an intuitive support portal
