Clients

Starburst lets you query all the connected data sources with the most widely used and best-supported query language — standard SQL. You can even query multiple data sources with the same query. Both simple queries and complex, federated queries all access the data on the source platform. There is no need for ETL processes.

    • You can use any supported client tool to connect to your Starburst Galaxy, Starburst Enterprise platform (SEP), and Trino cluster with some basic connection information. Many tools allow you much more powerful usage than simply running custom written queries. You can create complex reports, charts, dashboards, and many other useful results.

    The JDBC driver, ODBC driver, and Python clients allow you to connect many other applications. Use the Trino command line interface (CLI) for shell scripts and manual execution using a terminal, the built-in query editors in Starburst Galaxy or Starburst Enterprise platform (SEP) for a web-based IDE directly inside Starburst products.

    Select from the Starburst-supported clients on this page to query data in Starburst Galaxy or Starburst Enterprise, or view our machine learning page to learn how Starburst Galaxy and Starburst Enterprise can integrate into your existing machine learning workflows.

    In addition to Starburst-supported clients, many community-supported clients are also available from the Trino project, including:

    • Apache Airflow
    • Apache Dolphin Scheduler
    • Go client
    • Node.js client
    • Ruby client
    • VSCode
    • IBM Cognos Analytics
