Access control overview#

Starburst Enterprise platform (SEP) and the included enhanced connectors enable you to control access to the data exposed and accessible by SEP and the configured catalogs. The access control systems all follow role-based access control mechanisms with users, groups, roles, privileges and objects.

The access control/authorization set up needs to be chosen so that it is compatible with the used authentication system.

SEP includes the security features from PrestoSQL, such as different system access control methods, including the simple file based system access control.

SEP provides additional features suitable for different levels and mechanism for access control:

  • User impersonation

  • Password credential pass-through

  • Kerberos credential pass-through

  • Ranger integration

    • Global access control with Apache Ranger

    • Hive access control with Apache Ranger

    • Global access control with Privacera Platform

    • Hive access control with Privacera Platform

  • Hive access control with Apache Sentry

Availability varies based on the connector in use.
