    Try Starburst Enterprise #

    Starburst makes it easy to set up trial configurations of Starburst Enterprise platform (SEP).

    Do I need a license? #

    For evaluation purposes, SEP can be installed without a license. To use more than the basic functionality you must request a trial license.

    Common activities requiring a license include:

    • Insights Worksheets SQL editor and query history
    • Performance testing
    • Advanced authentication methods
    • Access control with Privacera and Ranger
    • Materialized views

    Select an evaluation method #

    Depending on the scope of your exploration, you can try SEP with our fully-fledged Kubernetes deployment in many major cloud services, or by using installation packages on bare metal.

    Robust proofs-of-concept #

    For anything from a simple exploration with a single coordinator and worker to a complex proof-of-concept with full performance and security, you can deploy and run SEP in the following cloud services using Helm charts with our Kubernetes-based deployment:

    • Amazon Elastic Kubernetes Service (EKS)
    • Google Kubernetes Engine (GKE)
    • Microsoft Azure Kubernetes Service (AKS)
    • Red Hat OpenShift 4.x or higher

    Our Helm charts are available directly through our Harbor instance, or through supported cloud providers’ marketplaces.

    Get started now.

    Limited evaluations #

    To evaluate SEP on any local or cloud-based resource using Linux, including a personal workstation, you have several options:

    • For RPM-based Linux distros, use the rpm package.
    • For any Linux distro, set up SEP with the tar.gz installation.
    • For any Linux distro, set up SEP as a cluster of Docker containers.

    These containers and packages are provided to allow you to gain initial familiarity with SEP concepts and practices on local or cloud machines that you control. While it is possible to configure, deploy, and manage a working cluster with these evaluation methods, we strongly discourage attempting that. SEP is an enterprise-grade, clustered application that requires a complex deployment strategy and extensive DevOps investment when deployed without Kubernetes. After completing this limited evaluation, use our K8s deployment with Helm charts to continue a more robust, production-level proof-of-concept.

    For initial evaluation only, you can use either method on recent releases of macOS. MacOS is explicitly not supported to run SEP services for any other purpose.
