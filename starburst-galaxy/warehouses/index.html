    Warehouses #

    A warehouse in Starburst Galaxy provides the resources to run queries against numerous catalogs. You can access the data exposed by the catalogs with Worksheets or other clients.

    Access your warehouses at any time by clicking Warehouses on the left hand menu. A demo warehouse is included by default.

    Starburst Galaxy allows you to create, edit, delete, start, and stop warehouses.

    Concepts #

    Creating and managing warehouses is an essential task for a platform administrator in Starburst Galaxy. A warehouse with the desired catalogs is required for a data consumer to use SQL statements in client tools to analyze the available data. The following concepts are important to perform this work efficiently.

    Cloud provider region and catalogs #

    Catalogs define the details to access a data source. Any data source is located in a specific cloud region of a specific cloud provider. For example, your Cloud SQL for MySQL database is hosted in the us-east1 region of Google Cloud.

    A warehouse can include one or more catalogs. If multiple catalogs are configured, you can query them with SQL using the same client connection. You can also query the data in multiple catalogs within one SQL statement.

    A warehouse and all its configured catalogs must be located in the same cloud provider and region. This allows for maximum performance and avoids data transfer costs for access across regions or even cloud providers.

    Size #

    The size of a warehouse is determined by the number of server nodes used to process queries, as well as the size of the individual nodes. A larger warehouse, consisting of more and larger nodes, is capable of processing more complex queries, handle more concurrent users, and provide higher performance, by using more resources.

    The available sizes include free, x-small, small, medium, large, x-large, and ``2x-large`. You can create a cluster with any size, and change size based on the current needs.

    Status and transitions #

    Any warehouse can be in one of the following three states:

    Stopped
    A stopped warehouse consists of a small configuration set only. No significant resources are used, and no costs are incurred.
    Running
    A running warehouse consists of a number of server nodes. It continues to be in the running state, while users are submitting queries for processing.
    Suspended
    A suspended warehouse consists of a small configuration set, and a mechanism to listen to incoming user request. It does not include any actively running server nodes, and no costs are incurred.

    A newly created warehouse begins in the stopped state, and can be started in the list of warehouse.

    A running warehouse can be manually stopped in the list of warehouse.

    A running warehouse transitions to the suspended status when no user queries are processed, and the configured time for Auto suspend elapsed. When a user submits a query to a suspended warehouse, the warehouse is started, and the query is processed. The user has to wait for the warehouse to start, which typically takes between one and five minutes.

    List of warehouses #

    The list of warehouses is accessible by clicking Warehouses on the left hand menu. It displays the following information about each warehouse:

    • to access a drop down of more actions including Edit warehouse, Open and Delete warehouse.
    • Name of the warehouse, used to identify a warehouse in the user interface as well as in the connection string for clients.
    • Status includes the current status as well as buttons to change the status including Stop and Start.
    • Query button to access Worksheets in the context of the warehouse.
    • Get connections shows details to use to connect clients to the warehouse.
    • Size displays the configured warehouse size.
    • Auto suspend displays the configured time for an inactive warehouse to transition to suspended status.
    • to edit the warehouse.

    Create a warehouse #

    Make sure that you have configured the desired catalogs to avoid restarts before creating a warehouse. Use the Create warehouse button above the list of warehouses and proceed with the following steps:

    1. Set the Warehouse name to a meaningful name for users.
    2. Configure the Warehouse size from the available sizes.
    3. Add one or more Catalogs to provide access to the configured data sources to users in this warehouse. The catalogs all needs to be using the same cloud provider and region as the warehouse itself.
    4. Choose the Cloud provider region to use for deploying and running the warehouse.
    5. Decide on the roles to grant access to the warehouse and configure access with Allowed roles.
    6. Click Create warehouse to save the configuration to the list of warehouses as a stopped warehouse.

    If desired, proceed to start the warehouse.

    Start a warehouse #

    Use the Start button in the list of warehouses.

    Stop a warehouse #

    Use the Stop button in the list of warehouses. You need to stop a warehouse to be able to delete it.

    Edit a warehouse #

    Use the Edit warehouse option available with the button, or the button in the list of warehouses to access the edit view.

    You can change the configuration of a warehouse without affecting your users and any running queries. Many changes, such as adding catalogs, changing size, or changing cloud region, require a new warehouse.

    Use the Update button in the list of warehouses to apply the configuration changes. The new warehouse is created transparently and new queries are transferred to it as soon as it is ready. At the same time running queries continue to operate and complete on the existing warehouse.

    Delete a warehouse #

    You need to stop the warehouse in the list of warehouses to be able to delete a warehouse. Once stopped, you can delete the warehouse with the Delete warehouse option in the drop down from the list. Alternatively, you can access the edit view for the warehouse and use the Delete warehouse button at the bottom of the view.
