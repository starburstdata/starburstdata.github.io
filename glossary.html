    Glossary #

    Terms A-E #

    Catalog
    Catalogs define and name the configuration to connect to, and query a data source.
    Cluster
    A cluster provides the resources to run queries against numerous data sources. Clusters define the number of workers, the configuration for the JVM runtime, configured data sources, and others aspects.
    Connector
    Transforms the underlying data into the SEP concepts of schemas, tables, columns, rows, and data types. A connector is specific to the data source it supports.
    COTS
    Common off-the-shelf. Refers to commodity hardware components.
    Data source
    A data source is a system where data is retrieved from. In SEP, you must connect to a data source so you can query from that source.

    Terms K-O #

    Marketplace
    Purchase a preconfigured machine images, containers and other needed resources to run SEP on their cloud hosts under your control.
    Marketplace: Amazon AWS
    A provider for all aspects of the required infrastructure. This includes using AWS CloudFormation for provisioning, Amazon Simple Storage Service (S3) for storage, Amazon Machine Images (AMI) and Amazon Elastic Compute Cloud (EC2) for computes, Amazon Glue as metadata catalog, and others.
    Marketplace: Google GCP
    Deploy in the Google Cloud Marketplace or using the Starburst Kubernetes solution on the Google Kubernetes Engine (GKE). GKE is a secure, production ready, managed Kubernetes service in GCP managing for containerized applications.
    Marketplace: Microsoft Azure
    Deploy using in the Azure Marketplace or using the Starburst Kubernetes solution onto the Azure Kubernetes Services (AKS). AKS is a secure, production-ready, managed Kubernetes service on Azure for managing for containerized applications.
    Marketplace: RedHat OpenShift
    A container platform using Kubernetes operators that automates the provisioning, management and scaling of applications to any cloud platform or even on-prem. Starburst Enterprise is available on RHM as of OpenShift version 4.

    Terms P-T #

    Persona: Administrator
    Owns platforms and services (ITIL-style). Has service SLA responsibility for the infrastructure supporting the cluster.
    Persona: Data consumer
    Owns data products such as reports, dashboards, models and the quality of analysis.
    Persona: Data engineer
    Owns schemas and is responsible for the source data quality and ETL SLA.
    Starburst Enterprise platform
    Helps companies harness the value of open source Presto, the fastest distributed query engine available today. Starburst adds connectors, security, and support that meets the needs for fast data access at scale.
    SEP
    Abbreviation of Starburst Enterprise platform.
    SQL
    Structured Query Language. The standard language used with relational databases.
    Trino
    Fast distributed SQL query engine for big data analytics. Formerly known as Presto.
