Starburst products provide a single platform to help you activate all the data in and around your data lake. Starburst products are built on Trino, the fastest open source, massively parallel processing SQL query engine designed for analytics of large datasets distributed over one or more data sources in object storage, databases and other systems.

    Query data #

    Learn about Trino SQL

    Starburst Galaxy and Starburst Enterprise bring you the power of Trino's SQL language. Trino SQL allows you to write queries in a single SQL variant to any supported data source. You can run both ad hoc and batch workloads with our massive parallel processing (MPP) engine, built for analyzing large amounts of distributed data with high concurrency.

    Querying data with Starburst

    Starburst brings all your data together in a single, federated environment. No more waiting for data engineering to develop complicated ETL.

    Video: Using SQL

    Trino’s SQL is ANSI-compliant and supports most of the SQL language features you depend on, so you can hit the ground running.

    Migrating analytics to Starburst

    Because not every data source implements the SQL standard the same, there could be slight differences between your data sources’ native SQL and SQL in Starburst products. Learn about migrating your existing tools and workflows to Starburst.
